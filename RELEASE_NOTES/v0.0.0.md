# Ai-Guard v0.0.0 — Pre-release

**Tag date:** 2026-06-30

First tagged pre-release before public launch. Freezes the current product state
so host-app integration (e.g. Jewgo event intake) can proceed against a known
baseline.

## Current capability

```text
policy → explain → validate → test-policy → request inspection → usage summary → real app integration
```

## Highlights

- **Real app integration pattern** — [`docs/integrations/real-app-pattern.md`](../docs/integrations/real-app-pattern.md)
- **Event intake example app** — [`examples/event_intake_app/`](../examples/event_intake_app/)
- **Request correlation IDs** — `requestId` on success; `auditRequestId` on policy blocks
- **Host metadata in audit logs** — `metadata` on chat calls; visible in `GET /v1/requests/:id`
- **Integration debugging runbook** — [`docs/runbooks/integration-debugging.md`](../docs/runbooks/integration-debugging.md)

## Boundary

```text
Your app     → Who may use this feature? (RBAC, business rules)
Ai-Guard     → Is this AI call allowed? (budget, model, safety, routing)
```

## Deploy notes

- Run migrations on deploy (through `0007_request_log_host_metadata.sql`).
- Log `ai_guard_request_id` alongside your domain entity ids.
- `metadata` is non-authoritative (logs/traces only).
